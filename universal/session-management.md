# Session Management — کم کردن توکن و انتقال context
> universal — برای همه پروژه‌ها
> updated: 2026-05-17

---

## مشکل

مکالمه‌های طولانی = توکن‌های زیاد = کند و گران. راه‌حل: از git و یه فایل کوچک
به جای نگه داشتن تمام تاریخچه در context استفاده کن.

---

## لایه‌بندی context (از ارزان‌ترین تا گران‌ترین)

```
۱. git history     رایگان — همیشه fresh — منبع حقیقت
۲. CLAUDE.md       کوچک — قوانین ثابت پروژه
۳. dev-knowledge   on-demand — دانش cross-project
۴. HANDOFF.md      snapshot — وضعیت فعلی
۵. مکالمه کامل   گران‌ترین — باید از اینجا فرار کنیم
```

---

## شروع هر session جدید

```bash
# ۱. ببین اخیراً چی ساخته شده
git log --oneline -15

# ۲. چیزهای uncommitted هست؟
git status

# ۳. آخرین تغییرات چی بوده
git diff HEAD~1 --name-only
```

بعد `CLAUDE.md` و `HANDOFF.md` رو بخون. **همین کافیه** — نیازی نیست کل codebase اسکن کنی.

---

## HANDOFF.md — چیه و کجاست

هر پروژه باید یه `HANDOFF.md` در root داشته باشه. این فایل فقط چیزایی رو نگه میداره که **git نمیدونه**:

- الان کجای پروژه‌ایم (milestone فعلی)
- قدم بعدی چیه
- تصمیم‌های معماری که in-code مشخص نیستن
- باگ‌های کشف‌شده

**چیزایی که نباید در HANDOFF باشن** (git بهتر میدونه):
- لیست components ساخته‌شده → `git log`
- چه فایل‌هایی تغییر کردن → `git diff`
- تاریخچه کار → `git log --oneline`

---

## آپدیت HANDOFF.md

بعد از هر milestone مهم (صفحه جدید، feature بزرگ، refactor):
1. بخش «الان کجاییم» آپدیت کن
2. قدم‌های بعدی آپدیت کن
3. **همراه بقیه تغییرات commit کن** — HANDOFF باید همیشه در git باشه

---

## وقتی session بزرگ شد — پیشنهاد session جدید

وقتی یکی از اینا اتفاق افتاد، به کاربر پیشنهاد بده session جدید شروع کنن:
- بعد از ۳-۴ feature/صفحه بزرگ
- وقتی جواب دادن به سوال‌ها کند شده (نشانه context پر شده)
- قبل از شروع یه بخش کاملاً جدید پروژه

پیام پیشنهادی:
> «مکالمه بزرگ شده. پیشنهاد میکنم HANDOFF.md رو commit کنیم و یه session جدید شروع کنیم.
> در session جدید فقط CLAUDE.md و HANDOFF.md رو می‌خونم و از همینجا ادامه میدم.»

---
